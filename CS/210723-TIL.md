### C언어 메모리 , 변수 종류 , 함수 호출 방식 (매우 중요)
```
- 프로그램 메모리 주소
- 컴퓨터에서 프로그램이 실행되기 위해서는 프로그램이 메모리에 적재(Load) 되어야 한다.
- 이 때, 프로그램의 크기를 충당할 수 있을 만큼의 메모리 공간이 있어야 한다.
- 일반적인 컴퓨터의 OS는 메모리 공간을 네 가지로 구분하여 관리한다.

- 메모리 영역
1. Code 영역 : 소스 코드
2. Data 영역 : 전역 변수, 정적 변수
3. Heap 영역 : 동적 할당 변수
4. Stack 영역 : 지역 변수, 매개 변수
```

### 변수의 종류
```
1. 전역 변수 (Global Variable)
: 프로그램의 어디서든 접근 가능한 변수
: main 함수가 실행되기도 전에 프로그램 시작과 동시에 메모리에 할당된다.
: 프로그램의 크기가 커질수록 전역 변수로 인해 프로그램이 복잡해질 수 있다.
: 메모리의 Data 영역에 적재된다.

2. 지역 변수 (Local Variable)
: 프로그램에서 특정한 블록에서만 접근할 수 있는 변수
: 함수가 실행될 때마다 메모리에 할당되어 함수가 종료되면 메모리에서 해제된다.
: 메모리의 Stack 영역에 적재된다.

3. 정적 변수 (Static Variable)
: 특정한 블록에서만 접근할 수 있는 변수
: 프로그램이 실행될 때 메모리에 할당되어 프로그램이 종료되면 메모리에서 해제된다.
: 메모리의 Data 영역에 적재된다.
: 변수 타입 앞에 static을 붙인다.

 블록에서만 접근이 가능하고 생성과 종료시점이 지역 변수와는 다르고 전역 변수와 같다.

4. 레지스터 변수 (Register Variable)
: 메인 메모리 대신 CPU의 레지스터를 사용하는 변수
: 레지스터는 매우 한정되어 있으므로 실제로 레지스터에서 처리될 지는 장담할 수 없다.
: 변수 타입 앞에 register를 붙인다.
```

### 함수의 호출 방식 2가지 (중요)
```
1. 값에 의한 전달 (Call by value)
: 값에 의한 전달 방식은 단지 값을 전달하므로 함수 내에서 변수가 새롭게 생성되는 것과 같다.
: 원래 변수의 값에는 영향을 미치지 못한다.

2. 참조에 의한 전달 (Call by reference)
: 참조에 의한 전달 방식은 주소를 전달하므로 원래의 변수 자체에 접근할 수 있습니다.
: 원래 변수의 값에 접근하여 값을 변경할 수 있다.
```
