# 2021.7.29. TIL (OS)


### OS - History로 보는 운영체제 (1950 ~ 2000년대)
```
1950년대
- ENIAC
 : 첫 번째 컴퓨터
 : 이 때는 OS가 없었다.
 : 1개의 응용프로그램을 실행시키는 것도 쉽지 않았음
 : 응용프로그램이 시스템 자원을 직접 제어


1960년대 초기
: 프로그램 종류도 많아지고, 사용자도 슬슬 많아지기 시작하는 시기

- 배치 처리 시스템 출현 (Batch processing system)
 : 여러 프로그램을 등록시켜 놓으면, 순차적으로 실행하는 시스템
 : 배치 처리 시스템을 기반으로 운영체제가 출현!!!!!!
 : 프로그램 실행시간에 따라 다음 프로그램의 시작시간이 달라진다. (꽤나 문제가 있었을 듯...)

- 배치 처리 시스템의 단점
 1. 컴퓨터 응답 시간(response time)이 오래 걸릴 수 있다.
 2. 실행 시간도 오래 걸릴 수 있다. (CPU가 필요없음에도 CPU를 점유할 수 있기 때문에)

-> 그래서 1960년대 후반에 시분할 시스템과 멀티 태스킹이 등장


1960년대 후반
: 새로운 개념이 제안됨 (실제 구현은 안됨)
 -> 시분할 시스템 (TIme sharing system)
 -> 멀티태스킹 (Multi Tasking)

1. 시분할 시스템 (TIme sharing system)
 : 응용프로그램이 CPU 사용 시간을 잘게 쪼개서, 여러 개의 응용프로그램을 동시에 실행하는 기법
 : 다중 사용자를 지원
 : 컴퓨터 응답 시간을 최소화

2. 멀티태스킹 (Multi Tasking)
 : 단일 CPU에서, 여러 응용프로그램의 병렬 실행을 가능하게 하는 시스템 (마치 동시에 여러개의 프로그램이 실행되는 것처럼 보인다.)
 : 가능한 CPU를 많이 활용하도록 하는 기능
(시간 대비 CPU사용률을 높임)


1970년대
: 제대로된 운영체제가 나왔다.

- Unix OS
 : 미국 AT&T사의 벨 연구소에서 개발
 : 켐 톰슨, 데니스 리치
 : 데니스 리치 (C언어로 개발)

- Unix OS 특징
 1. 현대 운영체제의 기본 기술을 모두 포함한 최초의 운영체제
 -> 멀티태스킹, 시분할 시스템, 멀티 프로그래밍, 다중 사용자 지원 등

Q. 운영체제 개발에 C언어가 필요했던 이유는?
1. 1970년대 이전
 : Assembly 언어로 SW 개발
 : 컴퓨터의 호환성 문제때문에 각각 다시 개발해줘야 했었다.
 : 프로그래밍 복잡도가 높음

2. 1970년대 C언어가 개발된 후
 : 컴파일러 - 다시 작성할 필요가 없음
 : 프로그래밍 복잡도가 낮음


1980년대
: PC 등장
: CLI -> GUI
: 1980년대 초반에는 아직 터미널 환경
: 1980년대 중후반부터 GUI환경이 사용   (Macintosh - 마우스)

1990년대
1. 응용프로그램 시대 by GUI 환경, PC 사용자 수 급증
2. Windows95 등장 -> Windows OS의 대중화
3. 네트워크 기술의 발전
 : www (월드 와이드 웹)의 대중화
 : 인터넷의 발전
4. 오픈 소스 운동 활성화 시작
 : Unix 계열 OS + 응용프로그램 자체 개발, 소스 오픈
 -> 이 때 Linux OS가 개발되었다!!!!

2000년대 이후
1. 오픈 소스 활성화
 : Linux OS
 : Apache (웹서버)
 : MySQL (DB)
2. 가상 머신, 대용량 병렬 처리 등 활성화  (빅데이터 처리)
````
